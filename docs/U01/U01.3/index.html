    Comentarios y separadores

    Comentarios - /* … */ (bloque) - // … (línea) - /** … */ (documentación)

    Separadores - () listas de parámetros (definición y llamada a métodos) - {} delimita bloques y valores iniciales de arrays - [] declaración de arrays y acceso a sus elementos - ; separa instrucciones - , separa identificadores/argumentos - . separa elementos de un objeto (acceso a miembros)

    2. Variables y tipos de datos

    Una variable permite almacenar datos, resultados e intermedios. Toda variable tiene asociado un tipo de datos que: - Define el conjunto de valores que puede almacenar. - Determina las operaciones que pueden realizarse.

    Declaración en Java 

    1
    Tipo Identificador;

    Tipos de datos

    • Primitivos
    • numéricos: byte, short, int, long, float, double
    • carácter: char
    • lógico: boolean
    • Referencia
    • String, arrays, clases e interfaces (se verán más adelante).

    Tipos de datos primitivos

    Numéricos - Enteros con diferentes tamaños (byte/short/int/long) y reales (float, double).

    Carácter | Nombre | Tamaño | Representa | |---|---|---| | char | 2 bytes | Caracteres (letras, números y símbolos). Java usa Unicode de 2 bytes. |

    Booleano | Nombre | Tamaño | Representa | |---|---|---| | boolean | 1 bit | Valores lógicos true / false. |

    Identificadores de variables I

    Los identificadores son los nombres de variables, clases, interfaces, atributos y métodos.

    Reglas - Java distingue mayúsculas y minúsculas: var1, Var1 y VAR1 son distintos. - Longitud prácticamente ilimitada. - No puede ser palabra reservada ni true/false. - No pueden duplicarse en el mismo ámbito. - Debe empezar por una letra; pueden usarse letras latinas/hebreas/cirílicas, y también $ (no recomendado) y _ (subrayado). - No usar espacios en blanco ni símbolos que coincidan con operadores.

    Identificadores de variables II (convenciones)

    • Variables y métodos: minúscula inicial.
    • Clases: mayúscula inicial.
    • Si el identificador tiene varias palabras: primera en minúscula (salvo clases), resto con mayúscula inicial (camelCase / UpperCamelCase).
    • variable: añoDeCreacion
    • clase: HolaMundo

    Inicialización de variables

    Si una variable no se ha inicializado, tiene un valor por defecto (cuando es miembro de clase): - Numéricos → 0 - char → '\u0000' - boolean → false - Referenciales (objetos) → null

    Nota: Las variables locales deben inicializarse antes de usarse; no tienen valores por defecto.

    Ámbito de una variable

    • El ámbito es la parte del programa donde una variable es visible y puede usarse.
    • Variables locales: declaradas dentro del cuerpo de un método; visibles solo en ese método.
    • Pueden declararse en cualquier punto del cuerpo, aunque es buena práctica al principio.
    • También pueden declararse dentro de un bloque { … }; serán visibles solo dentro del bloque.
    • Las variables definidas en un bloque deben tener nombres diferentes.

    Constantes

    Para declarar una constante se usa el modificador final.

    3. Literales

    Literales I

    Definición: un literal es un valor que se expresa a sí mismo.

    Enteros - Decimal (base 10): 21 - Octal (base 8): 025 - Hexadecimal (base 16): 0x3A - Sufijo L/l para long: 21L

    Reales - Notación decimal: 345.678, 0.00056 - Notación científica/exponencial: 3.45678e2, 5.6e-4 - Sufijos: F/f → float, D/d → double - Por defecto es double; para float usar sufijo F.

    Literales II — carácter

    • Entre comillas simples '…'.
    • Ejemplos: 'a', 'B', '{', 'ñ', 'á'.
    • Código Unicode del carácter (octal/hexadecimal):
    • '\141' (octal) → 'a'
    • '\u0061' (hex) → 'a'
    • Secuencias de escape más comunes:
    • \' comilla simple
    • \" comillas dobles
    • \\ barra invertida
    • \b backspace
    • \n salto de línea
    • \r retorno de carro
    • \t tabulador

    Literales III — booleano y String

    • Booleano: true, false.
    • String: entre comillas dobles "…". 
      1
2
      System.out.println("Primera línea\nSegunda línea del string\n");
System.out.println("Hol\u0061"); // "Hola"

    Conversión explícita (casting)

    También mediante funciones de librerías cuando corresponda.

    Prioridad de operadores I

    Cuando hay varias operaciones en una expresión, se evalúa con un orden predeterminado: 1. Paréntesis (de dentro a fuera) 2. Unarios 3. Aritméticos - Multiplicativos: *, /, % - Sumativos: +, - 4. Comparativos/Relacionales 5. Lógicos - ! (NOT) - && (AND) - || (OR) 6. Asignación

    Cuando aparecen operadores de la misma prioridad, el compilador evalúa de izquierda a derecha.

    Son conjuntos de valores constantes para los que no existe un tipo predefinido (por ejemplo: días de la semana, estaciones, meses).

    Constantes - Math.E = 2.7182818284590452354 - Math.PI = 3.14159265358979323846

    Sistema de tipos

    Dos categorías generales: - Tipo simple o valor: la variable contiene directamente sus datos; cada variable tiene su propia copia. - Tipo referencia: la variable contiene una referencia/puntero al objeto; dos variables pueden referirse al mismo objeto.

    Los tipos simples (primitivos) tienen clases envoltorio equivalentes: - byte → java.lang.Byte - short → java.lang.Short - int → java.lang.Integer - long → java.lang.Long - float → java.lang.Float - double → java.lang.Double - char → java.lang.Character - boolean → java.lang.Boolean

    Nota: existen además tipos comunes, cadenas, matrices (arrays), clases y enumeraciones definidas por el usuario.
