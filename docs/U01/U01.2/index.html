    ¿Qué es un programa?

    Una serie de instrucciones dadas al ordenador en un lenguaje que entiende, para indicarle exactamente lo que queremos que haga.

    Si el ordenador no entiende alguna instrucción, normalmente lo comunicará mediante mensajes en pantalla.

    Tipos de lenguajes

    Lenguaje Máquina

    • Lenguaje que entiende directamente el computador.
    • Instrucciones expresadas en alfabeto binario.
    • Los datos se referencian por direcciones de memoria (no hay nombres).
    • No pueden incluirse comentarios.
    • Depende del procesador del ordenador → un programa en lenguaje máquina para un procesador no se ejecuta en otro distinto (baja portabilidad).

    Ensamblador

    • Primer intento de sustituir el lenguaje máquina por uno más cercano al nuestro.
    • Relación aproximada 1:1 entre instrucción ensamblador e instrucción máquina.

    Lenguajes de alto nivel

    • Independientes de la arquitectura del computador.
    • Una instrucción de alto nivel suele traducirse a varias instrucciones de lenguaje máquina.

    Traductores

    Compiladores

    • Traducen completamente un programa fuente (alto nivel) generando un programa objeto (lenguaje máquina) semánticamente equivalente.
    • Informan de los errores existentes en el programa fuente y solo generan el objeto si no hay errores.
    • El programa objeto se almacena en un fichero que puede ejecutarse sin volver a traducir.

    Intérpretes

    • Permiten que un programa fuente se vaya traduciendo y ejecutando directamente sentencia a sentencia.
    • Captan una sentencia, la analizan y la interpretan para su ejecución inmediata.
    • No crean ningún fichero objeto.

    Proceso de compilación

    • A partir del programa fuente se ejecuta el compilador, que verifica y traduce el código.
    • Se genera un programa objeto y, opcionalmente, un ejecutable.
    • En Java, la salida del compilador son bytecodes (ficheros .class).

    Ciclo de vida de una aplicación informática

    Proceso desde el planteamiento del problema hasta tener una solución instalada y en funcionamiento por el usuario, mientras sea de utilidad.

    Fases principales (agrupadas):

    Fases de análisis y diseño 1. Análisis → Especificaciones (E/S) 2. Diseño → Algoritmo 3. Programación → Codificación

    Construcción 4. Edición → Programa fuente 5. Compilación → Programa objeto

    Instalación 6. Montaje → Programa ejecutable 7. Prueba de ejecución → Aplicación lista

    Explotación y mantenimiento 8. Uso en producción, corrección y evolución.

    ¿Qué es Java?

    • Lenguaje de propósito general válido para aplicaciones profesionales.
    • Programas ejecutables (bytecodes) independientes de la arquitectura.
    • Se ejecutan en múltiples equipos y sistemas operativos mediante la JVM.

    Características del lenguaje

    • Orientado a objetos intrínsecamente.
    • Funciona perfectamente en red.
    • Aprovecha características de lenguajes modernos evitando inconvenientes (en particular de C++).
    • Gran funcionalidad gracias a librerías (clases).
    • Sin punteros manejables por el programador (gestionados internamente de forma transparente).
    • Gestión automática de memoria (garbage collector).
    • Facilita generar aplicaciones con menos errores.
    • Multithreading: ejecución concurrente de tareas.
    • Puede considerarse como una evolución de C++.

    ¿Compilado o interpretado?

    • Aunque estrictamente interpretado, necesita un proceso previo de compilación.
    • La compilación crea bytecodes (pseudocódigo cercano a máquina) en ficheros .class.
    • La JVM interpreta/ejecuta esos bytecodes en cualquier plataforma compatible.

    La Máquina Virtual Java (JVM)

    • Un mismo programa fuente, compilado en distintos sistemas, genera el mismo bytecode.
    • La JVM traduce el bytecode a código nativo de la máquina donde se ejecuta (just-in-time / al vuelo).
    • Existe una JVM distinta por plataforma (Windows, Linux, macOS, etc.).
    • Se carga en memoria y va traduciendo dinámicamente; no ocupa mucho espacio.

    Entorno de desarrollo JDK (Java Developer’s Kit)

    Herramienta básica para desarrollar aplicaciones o applets en Java.

    Incluye, para línea de comandos:

    • Compilador: javac
    • Intérprete: java y javaw
    • Depurador: jdb
    • Visualizador de applets: appletviewer
    • Generador de documentación: javadoc
    • Desensamblador de clases: javap
    • Generador de cabeceras para clases nativas en C: javah

    En Windows el instalador crea la estructura de directorios correspondiente.

    IDE vs. editor de código

    IDE (Integrated Development Environment): integra todo lo necesario para desarrollar de forma cómoda. Ejemplos: IntelliJ IDEA, NetBeans, Eclipse, BlueJ.

    Editor de código: herramienta ligera para escribir código sin funciones avanzadas. Ejemplos: Visual Studio Code, Notepad++, Sublime Text, bloc de notas.

    Fichero fuente y convención de nombres

    • Extensión de los ficheros de código fuente en Java: .java.
    • Convención UpperCamelCase (sin acentos, espacios, guiones bajos ni números iniciales). Ejemplos: Prova.java, HolaMundo.java, ElMeuPrograma.java.
    • El nombre de la clase debe coincidir exactamente (incluyendo mayúsculas/minúsculas) con el nombre del fichero.
    • Java es sensible a mayúsculas: HolaMundo, holaMundo y Holamundo son distintos.

    Explicación

    1. Declaración de clase: class HolaMundo { ... }.
    2. Todo el código en Java está formado por clases.
    3. La palabra reservada class seguida del nombre de la clase; el contenido va entre llaves { ... }.

    4. El nombre de la clase coincide con el nombre del fichero HolaMundo.java.

    5. Método principal: public static void main(String[] args) { ... }.

    6. Es el punto de entrada de la aplicación.
    7. public y static son palabras reservadas que se explicarán con POO.

    8. Parámetro String[] args: conjunto de cadenas; el nombre args es por convención.

    9. Salida por pantalla: System.out.println("¡Hola, mundo!");.

    10. Usa el método println() del espacio de nombres System.out para imprimir con salto de línea.
