    • Se utiliza para una decisión true o false
    • Si la condición es true, se ejecutan las instrucciones siguientes

    Uso de if...else

    • Decisión con dos opciones como mínimo
    • Si la condición es true, se ejecutan las instrucciones tras if
    • Si la condición es false, se ejecutan las instrucciones tras else
    1

    System.out.println("No puedes salir a la hora del patio");

    System.out.println("Puedes salir");
}

    Detalles de if...else

    • La parte else es opcional
    • El bloque requiere {} si contiene más de una instrucción
    • Un bloque puede ser la sentencia vacía ; (no se ejecuta nada)

    Instrucción switch...case

    • Sintaxis compacta para seleccionar un bloque según un valor
    • Alternativa a if...else anidados
    • Funciona sobre enteros, booleanos o caracteres
    • Las condiciones deben ser excluyentes

    switch (mes) {
   case 1:
       print("enero");
       break;
   case 2:
       print("febrero");
       break;
   // ...
   default:
       print("no sé");
}

    Detalles de switch...case

    • Las sentencias break terminan la ejecución del bloque
    • Si no hay break, el código siguiente se ejecuta
    • El default es opcional

    Directrices para elegir una estructura de decisión

    • if: controlar la ejecución de un único bloque de código
    • if...else: controlar dos secciones mutuamente excluyentes
    • if...else if...: controlar múltiples condiciones excluyentes de forma secuencial
    • switch...case: lista de valores posibles

    Uso de while

    • Ejecuta el código solo si la condición es true
    • Se repite hasta que la condición sea false
    • La condición se evalúa al principio

    Detalles de for

    • El contador es una variable local al bucle
    • Los componentes sintácticos (inicialización, condición y actualización) pueden omitirse si no son necesarios, pero los puntos y comas deben mantenerse. Por ejemplo:
    1

   // bucle infinito
}

    break y continue

    • break: termina el bucle
    • continue: comienza una nueva repetición, omitiendo el resto del bloque actual

    Se recomienda no usar break y continue salvo que sea evidente su necesidad

    for (int i = 0; i < 10; i++) {
   if (i == 5)
      break; // Sale del bucle cuando i es 5
   if (i % 2 == 0)
      continue; // Salta a la siguiente iteración si i es par
   System.out.println(i); // Solo imprime valores impares menores que 5
}

    Directrices para elegir una estructura de repetición

    • while: cuando no se conoce el número de iteraciones y se desea repetir mientras se cumpla una condición.
    • do...while: similar a while, pero garantiza al menos una iteración.
    • for: cuando se conoce el número exacto de iteraciones.
    • foreach: para iterar sobre elementos de una colección o array (lo veremos más adelante).
